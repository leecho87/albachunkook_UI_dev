웹표준가이드 명명규칙

1. 명명규칙
  • 영문자로 시작하며 한글은 사용하지 않는다. 공백은 멀티클래스 사용을 제외하고는 사용하지 않으며, 의미가 명확한 클래스명을 사용한다. lnb나 gnb같은 널리 사용하는 줄임말을 제외하고는 무분별한 줄임말 (ex : sb1, sb2) 은 지양하며, 서비스나 페이지, 요소의 특성 등을 고려하여 명확하게 사용하도록 한다. 이는 명확한 네이밍을 통해, 누가 봐도 명확하게 의미 전달을 하기 위함이다.
2. 마크업 명명규칙
  • 이 단락에서는, 문서 내에 사용하는 class(클래스)의 명명규칙을 다룬다.
    아이디는 특수한 경우를 제외하곤 사용하지 않는다. 클래스는 항상 소문자 (ex : header, container, contents, section, footer 등)로 시작한다. B.E.M(Block__Element--Modifer)의 규칙에 따라 언더스코어(_)와 하이픈(-) 을 활용하여 작성한다.
    위에서 언급한 대,소문자를 활용한 방식을 각각 파스칼 표기법, 낙타 표기법이라 지칭하며 각각은 다음과 같다.
3. 명명 모델 규칙
  • 웹표준파트에서 지향하는 네이밍기법은 B.E.M 기법 Block,Element,Modifier이다.
    B.E.M기법은 Block 단위로 각각을 Article처럼 어디에서든 활용하는 장점이 있다. 아래의 설명이 B.E.M모델의 예제이다.
  • B.E.M 예제 확인하기

    <Block>
    .human .login <Block + Element>
    .human__am .login__id <Block + Element + Modifier>
    .human__am--left .login__id--active
