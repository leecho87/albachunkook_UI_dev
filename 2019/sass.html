웹표준가이드 SASS 규칙

1. SASS란?
  • css의 전처리기로써, css에서 불가능한 mixin,include,extend,연산 등이 가능하여 css문서를 유연하고 빠르게 할 수 있도록 도와주는
    전처리기(pre-process)이다.
2. SASS를 써야하는 이유
  • - 구성원들끼리의 규칙(컨벤션)을 통해 효율적인 코드 작성 가능
  • - sass 문법으로 작성된 부분에 대한 css selector의
    시각적인 상속관계 파악 가능
  • - css에서의 변수 및 함수 사용 가능
  • - 웹 사이트의 용량이 방대해짐에 따른 코드의 효율적인 관리가 필요

웹표준가이드 SASS 규칙 - 특징, 장점

3. 특징
  • - Less보다 더 오래된 전처리기
  • - 해외 뿐만이 아닌 "국내에서도 대중화" 되어있는 전처리기
  • - "Ruby" 언어를 기반으로 동작
  • - 기존 문법과의 호환성(if, each, for, while)
4. 장점

  • 해외 뿐만이 아닌 "국내에서도 대중화" 되어있는 전처리기

6. @mixin, include
  • - @mixin 과 include 방식으로 작업하며 css파일명과 폴더명을 똑같이 만들고 preset, util로 구분하여 정의한다.
  • - @mixin은 변수로써, @mixin variable(a,b,c){ ... } 방식으로 선언한다.
  • - 선언한 값은 .header-wrapper { @include variable } 로 적용한다.
  • - 기존 문법과의 호환성(if, each, for, while)

  • - sass는 gulp-sass를 사용하며, task에 대한 자세한 정보는 style.task.js 와 gulp.config.js를 참조하면 된다.
  • 자세한 설명 보기

    현재는 4개의 task로 구성하였으며,
    style:clean -> CSS Path의 파일을 삭제한다.
    style:work -> 테스트서버 css 파일을 compile 한다.
    style:deploy -> 압축버전의 css을 compile한다
    style:deploy-override -> 배포폴더를 만들었으나 배포를 안했을 경우, 덮어씌운다.
